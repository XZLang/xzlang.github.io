这里filename就摆在这里，所以先尝试一波任意文件下载的操作，这里需要跳到上层目录才可以下载文件，所以构造如下，成功下载源码。

然后把其他的文件都给下载下来

一共这些，然后一个一个开始分析，这里最重要的就是class.php文件

参数绑定的SQL语句，基本上不考虑注入了。先找可能存在的漏洞点，我们可以看到在File类中的close方法中有一个熟悉的file_get_contents方法，可以进行文件的读取,如果能触发该方法，就有可能获取flag。FileList类中存在着一个call魔术方法，而且FileList类没有close方法，根据call方法的内容，我们只要让一个FileList的对象调用close，就会触发File类的close方法，造成文件读取，然后内容通过FileList的析构方法打印出来。

即$user->__destruct() => $db->close() => $db->__call('close') => $file->close() => $results=file_get_contents($filename) => FileList->__destruct()输出$result。

然后构造phar的基操pop链一把梭

运行这个PHP文件，生成了一个phar文件，这个文件改后缀名为gif（因为phar伪协议解析的时候会将GIF同样当作phar来解析，以此绕过上传限制），上传之后，删除该文件，抓下删除的包，改文件名为phar://phar.gif

就可以拿到flag。
